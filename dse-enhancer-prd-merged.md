## 1. 产品定位

### 目标用户
- 在港就读的大陆学生
- DSE英语成绩普遍在2-3级，目标提升到3-5级
- 每天放学后可以抽出40分钟集中学习

### 核心痛点
1. 大陆学生基础不差，但不适应DSE的题型和出题思路
2. 香港本地培训班太贵，时间不灵活
3. 现有线上平台内容太泛，不针对DSE垂直场景
4. 缺乏个性化自适应，不知道每天该练什么

### 差异化
- ✅ 垂直只做DSE英语，完全贴合考纲
- ✅ **访客模式** - 用户不注册也能进行入学测试和体验学习，降低使用门槛
- ✅ AI自适应学习，每天自动安排针对性练习
- ✅ 一键开始，40分钟完成，碎片化设计贴合课后场景
- ✅ AI评分写作口语，即时反馈，不需要等老师批改
- ✅ 游戏化PK排行榜，激励坚持学习
- ✅ 大陆学生针对性弱项强化，完全不同于香港本地平台
- ✅ 繁简体切换，阅读体验友好
- ✅ 支持香港学制（P1-P6, F1-F6），Pre Level为小学生铺垫

## 2. 核心业务流程

### 整体流程

```
用户访问 → 【访客模式】→ 选择年级（P1-P6, F1-F6）→ 入学测试（15题）→ 能力定级显示
    ↓                                           ↓
    ↓                              进入每日学习（访客使用相同学习入口）
    ↓                                           ↓
    ↓                              首次学习体验 → 弹出注册引导（数据不保存）
    ↓                                           ↓
注册/登录 → 选择年级（P1-P6, F1-F6）→ 入学测试（15题）→ 能力定级
    → 输入目标等级 + 考试日期 → 生成初始学习计划
    ↓
每日访问 → 点击【一键开始今日学习】→ 系统自动组合内容 → 按顺序完成四个模块
    ↓
每日结算 → 各模块计分 → 生成总分（满分100）→ 分析弱项 → 调整明日练习 → 打卡完成
    ↓
持续迭代 → 能力提升（难度系数1.0→1.9）→ 达到目标等级 → 完成学习目标
```

### 访客模式说明

**年级选择（香港学制）**：
| 年级范围 | 对应学制 | 能力等级范围 |
|---------|---------|-------------|
| P1-P6 | 小学一至六年级 | Pre Level（预备级，不计入DSE 1-5级）|
| F1-F3 | 中一至中三 | Level 1-2（基础级）|
| F4-F6 | 中四至中六 | Level 2-5（DSE备考级）|

**注意**：P1-P6小学生使用Pre Level难度体系，为将来DSE学习打基础；F1-F6学生按DSE标准评定Level 1-5。

| 用户类型 | 可以做 | 不能做 |
| --- | --- | --- |
| 访客模式 | 选择年级、入学测试（15题）、查看能力等级、每日学习体验、查看当日得分 | 成绩持久化保存、自适应学习推送、错题本、能力图谱、排行榜、跨会话数据 |
| 免费注册（试用期内） | 全部功能，每日练习题量有限制 | 超出题量限制的内容 |
| 免费注册（试用到期） | 每日10题词汇、查看能力图谱、查看错题本 | 完整学习流程、AI写作/口语评分、PK对战 |
| 付费会员 | 全部功能，无限制 | — |

**访客模式流程**：
1. 访客进入后选择年级（P1-P6, F1-F6）
2. 进行入学测试（15题自适应测试），与注册用户相同
3. 测试完成后显示能力定级结果（P1-P6显示Pre Level，F1-F6显示Level 1-5）
4. 访客数据临时存储在localStorage，仅本次会话有效
5. 进入每日学习（使用与注册用户相同的入口和界面）
6. 学习过程中可查看当日得分（满分100分）
7. **首次学习结束后**：数据不保存，无法进行后续自适应学习
8. 体验结束后弹出注册引导

**访客数据存储**：
- 入学测试答案和结果 → localStorage（会话级）
- 单次学习数据 → 内存（页面关闭即丢失）
- 学习历史、错题本、能力图谱 → ❌ 不支持

**注册引导话术**：「注册后可保存学习进度，AI会根据你的弱项智能推送练习，帮你更快提分！当前进度将丢失，建议立即注册。」

### 关键设计：两种学习模式自由切换

#### 模式 1：轻松学（40分钟）
适合时间紧张的用户，快速练习核心内容。MVP阶段包含词汇、语法、阅读三个核心模块（听力、写作、口语在Part B或专项练习中提供）。

| 模块 | 用时 | 默认题量 | 具体内容 |
|------|------|----------|----------|
| 单词闯关 | 10分钟 | 15-20题 | 闯关背单词/生词本复习 |
| 词汇(Part A) | 10分钟 | 10-15题 | 词汇测试练习 |
| 语法(Part A) | 10分钟 | 10-15题 | 语法测试练习 |
| 阅读(Part A) | 10分钟 | 2-3篇 | 阅读测试练习 |
| **总计** | **40分钟** | **约40题** | MVP核心模块 |

**说明**：用户可在固定时间内完成更多题目，正确率达标可获额外加分（边际递减）。听力、写作、口语可在勤奋学模式的Part B或专项练习中进行。

---

#### 模式 2：勤奋学（60分钟）
适合想多学习的用户，在轻松学基础上增加Part B深度练习和错题复习。

| 模块 | 用时 | 说明 |
|------|------|------|
| 轻松学核心模块 | 40分钟 | 单词闯关 + 词汇 + 语法 + 阅读（每日必做）|
| Part B 补充 | 15分钟 | 专项深度练习（见下方轮换表） |
| 错题复习 | 5分钟 | 艾宾浩斯间隔重复复习 |
| **总计** | **60分钟** | 深度学习模式 |

**Part B 每周轮换安排**：

**勤奋学每周轮换安排**：

| 星期 | Part B 补充安排 |
|------|----------------|
| **周一** | 听力 Part B |
| **周二** | 词汇 Part B |
| **周三** | 语法 Part B |
| **周四** | 阅读 Part B |
| **周五** | 写作 Part B |
| **周六** | 口语 Part B |
| **周日** | 自由安排 | 无强制一键学习，周日休息 |

---

### 错题复习记录
- Part B 的错题也会同样记录在错题复习里
- 下次错题复习时会包含 Part B 的错题

### Part B 专项深度练
- 用户还可以去「专项练习」里完整练 Part A + Part B
- 周日可以自由安排，或者做一套完整模考

### 写作和口语 Part 设计说明

| 模块 | 日常打卡（一键学习） | 专项练习（自由练习） |
|------|-------------------|-------------------|
| **写作** | 只练 Part A（情景写作，200-300字，限时10分钟） | Part A + Part B（完整写作练习，无时间限制） |
| **口语** | 只练 Part A（个人独白，1分钟准备+2分钟发言，限时10分钟） | Part A + Part B（完整口语练习，无时间限制） |

日常打卡只练单个 Part，控制在10分钟内；专项练习可以练完整 Part A + Part B。

## 3. 功能模块详情

### 3.1 用户体系

#### 基础账号
| 功能 | 说明 |
|------|------|
| 注册登录 | 手机号 + 验证码登录（支持微信/Google快捷登录） |
| 个人档案 | 年级、目标分数、当前等级、学习天数、累计积分 |
| 能力图谱 | 可视化展示听/说/读/写四个维度的当前等级 |
| 会员系统 | 免费试用7天，付费会员解锁全部功能 |

#### 亲子账号体系（解决家长帮孩子注册问题）

**场景**：很多是家长付费，孩子使用，家长需要看进度

**方案设计**：
- 主账号：可以是家长注册，也可以是孩子自己注册
- 主账号可以**添加多个子账号**（对应多个孩子）
- 每个孩子一个独立子账号，有独立的学习进度、错题本、能力图谱
- 孩子登录使用子账号，做题练习
- 家长登录主账号，可以：
  1. 查看每个孩子的学习报告
  2. 管理会员订阅（一个会员可以支持多个孩子？或按孩子数收费）
  3. 接收每周学习汇总邮件

**账号关系图**：
```
主账号（家长/学生本人）
  ├─ 子账号 老大 → 独立学习数据
  ├─ 子账号 老二 → 独立学习数据
  └─ 子账号 老三 → 独立学习数据
```

**登录流程**：
- 家长扫码/输入验证码登录主账号
- 在主账号仪表盘选择要查看哪个孩子，或者切换到孩子账号学习
- 孩子也可以直接用子账号手机号登录（如果有自己手机）
- 支持快捷切换账号，不需要重新登录

**付费方案**：
- 基础会员：支持 1 个孩子
- 家庭会员：支持最多 3 个孩子（优惠价）
- 这样满足多孩家庭需求，也增加ARPU

### 3.2 入学测试

**测试目标**：根据学生年级和答题表现，评定当前能力等级，建立个性化学习起点。

#### 年级与能力等级对应

| 选择年级 | 学制 | 能力等级范围 | 测试题难度范围 |
|---------|------|-------------|---------------|
| P1-P3 | 小学低年級 | Pre Level (0.5-0.9) | 基础词汇、简单句型 |
| P4-P6 | 小学高年級 | Pre Level (0.7-1.0) | KET/PET水平 |
| F1-F3 | 中一至中三 | Level 1-2 (1.0-2.0) | DSE基础难度 |
| F4-F6 | 中四至中六 | Level 1-5 (1.0-5.0) | DSE全范围难度 |

#### 测试流程

```
1. 用户选择年级（P1-P6, F1-F6）
2. 15分钟，15题自适应测试
   - 起始难度根据年级设定
   - 答对 → 下一题难度提升（增幅0.1-0.2）
   - 答错 → 下一题难度降低（降幅0.1-0.2）
   - 连续答对/答错会加速调整
3. 计算最终等级：
   - 基于答题正确率和题目难度加权计算
   - 公式：等级 = 基准等级 + Σ(题目难度×答题结果权重)/15
4. 输出能力报告：
   - 当前等级显示（如：Level 2.3）
   - 各题型表现分析
   - 与目标等级的差距
5. 生成个性化学习路径（注册用户）
```

#### 能力等级体系（基于DSE官方标准）

**参考依据**：香港中学文凭考试（DSE）英语科水平参照，对应CEFR国际语言能力标准

**总等级范围**：Pre Level (0.5-1.0) → Level 1 (1.0-1.9) → Level 5 (5.0-5.9)

| 等级 | DSE官方 | CEFR对应 | 能力描述 | 适用年级 |
|------|---------|----------|----------|----------|
| **Pre Level**<br>0.5-1.0 | - | Pre-A1<br>~A1 | 基础入门：掌握基础词汇和简单句型，能进行最基本的日常对话 | P1-P6 |
| **Level 1**<br>1.0-1.9 | Level 1 | A2 | 初级水平：理解常用表达，能在熟悉情境下进行简单沟通，词汇量约1500-2000 | F1-F2 |
| **Level 2**<br>2.0-2.9 | Level 2 | B1 | 中级水平：理解一般性话题，能应对日常情境，词汇量约2500-3000 | F2-F3 |
| **Level 3**<br>3.0-3.9 | Level 3 | B2 | 良好水平：理解复杂文本，能流利表达观点，词汇量约3500-4500 | F3-F4 |
| **Level 4**<br>4.0-4.9 | Level 4 | C1 | 优秀水平：理解高难度文本，能灵活准确使用语言，词汇量约5000-6500 | F5-F6 |
| **Level 5**<br>5.0-5.5 | Level 5 | C1+ | 卓越水平：轻松处理复杂材料，表达精确流畅，词汇量约7000+ | F6 |
| **Level 5***<br>5.6-5.9 | Level 5* | C2 | 顶尖水平：接近母语者水平，能处理最复杂的语言任务 | F6冲刺 |

---

**各等级详细能力标准**：

### Pre Level (0.5-1.0) - 基础预备阶段
**词汇要求**：800-1200词
**语法能力**：
- 基础时态（一般现在时、一般过去时）
- 简单句型（主谓宾结构）
- 基本疑问句和否定句
**阅读能力**：理解50-80词的简单短文
**写作能力**：能写30-50词的简单句子和段落
**听力能力**：理解慢速、清晰的简单对话
**口语能力**：进行最基本的自我介绍和日常问候

### Level 1 (1.0-1.9) - DSE Level 1
**词汇要求**：1500-2000词
**语法能力**：
- 常用时态（现在/过去/将来时）
- 基础从句（that/which引导）
- 情态动词基础用法
**阅读能力**：
- 理解100-150词的短文
- 抓住主旨和基本事实
**写作能力**：
- 撰写80-120词的文章
- 基本段落结构（开头-主体-结尾）
- 简单连贯手段
**听力能力**：
- 理解熟悉话题的清晰慢速讲话
- 抓住关键信息
**口语能力**：
- 进行1-2分钟的简单独白
- 表达基本观点
- 有明显停顿和语法错误

### Level 2 (2.0-2.9) - DSE Level 2（合格线）
**词汇要求**：2500-3000词
**语法能力**：
- 全部时态的正确使用
- 条件句（第一、二条件句）
- 被动语态基础
- 定语从句和状语从句
**阅读能力**：
- 理解200-300词的中等难度文章
- 理解主旨、细节和隐含意义
- 识别作者态度和观点
**写作能力**：
- 撰写150-200词的文章
- 基本连贯性和组织结构
- 使用简单连接词（however, therefore, in addition）
- 有一些语法错误但不影响理解
**听力能力**：
- 理解标准速度的常速讲话
- 理解主要观点和重要细节
- 能听懂不同口音的基础内容
**口语能力**：
- 进行2分钟较连贯的独白
- 使用适当连接手段
- 词汇和语法有局限性
- 能应对日常交流情境

### Level 3 (3.0-3.9) - DSE Level 3（良好）
**词汇要求**：3500-4500词
**语法能力**：
- 所有时态和语态的灵活运用
- 第三条件句、虚拟语气
- 非谓语动词（不定式、分词、动名词）
- 倒装、强调句型
**阅读能力**：
- 理解300-500词的中高难度文章
- 分析文章结构和论证方法
- 理解复杂句子结构
- 批判性阅读
**写作能力**：
- 撰写200-300词的文章
- 清晰的逻辑结构和论证
- 使用多样连接手段（furthermore, consequently, notwithstanding）
- 词汇较丰富，语法错误较少
- 能处理多种文体（书信、报告、议论文）
**听力能力**：
- 理解各种口音的正常速度讲话
- 理解隐含意义和态度
- 处理复杂信息（讲座、讨论）
- 区分事实和观点
**口语能力**：
- 进行2-3分钟流利连贯的独白
- 使用丰富词汇和复杂句型
- 能详细阐述观点并举例
- 有自然的停顿和语调

### Level 4 (4.0-4.9) - DSE Level 4（优秀）
**词汇要求**：5000-6500词
**语法能力**：
- 复杂句型灵活准确运用
- 高级语法结构（独立主格、悬垂分词）
- 修辞性语言使用
- 语域（register）灵活运用
**阅读能力**：
- 轻松理解500+词的高难度文章
- 理解抽象和复杂概念
- 分析作者意图和修辞手法
- 快速阅读并抓住关键信息
**写作能力**：
- 撰写300-400词的高质量文章
- 逻辑严密，论证充分
- 使用高级词汇和复杂句型
- 风格正式、表达精确
- 能驾驭各种复杂文体
**听力能力**：
- 轻松理解各种复杂听力材料
- 处理学术讲座和专业讨论
- 理解隐含意义和文化内涵
- 快速适应不同口音
**口语能力**：
- 进行3分钟以上流利连贯的复杂表达
- 使用复杂语法和高级词汇
- 逻辑严密，观点清晰
- 能处理抽象和复杂话题
- 语音语调自然，接近母语者

### Level 5 (5.0-5.5) - DSE Level 5（卓越）
**词汇要求**：7000+词
**语法能力**：
- 语法使用接近完美
- 灵活运用各种复杂结构
- 语域转换自如
- 修辞运用娴熟
**阅读能力**：
- 轻松处理各类高难度文本
- 包括文学、学术、专业材料
- 深度理解和分析
**写作能力**：
- 撰写400词以上高质量文章
- 语言精炼、表达精确
- 逻辑严密、论证深刻
- 具有个人风格和说服力
**听力能力**：
- 轻松理解所有类型的听力材料
- 包括快速、带口音、专业内容
**口语能力**：
- 流利、准确、自然的表达
- 能处理任何复杂话题
- 具有说服力和感染力

### Level 5* (5.6-5.9) - DSE Level 5*（顶尖）
**词汇要求**：8000+词
**各项能力**：
- 接近母语者水平
- 能处理最复杂、最微妙的语言任务

#### 各技能维度详细标准（基于DSE官方评估框架）

根据HKEAA官方文件，DSE英语科从四个维度评估学生能力：

**维度1：阅读理解 (Reading)**

| 等级 | 文本复杂度 | 理解能力 | 推理能力 | 词汇处理 |
|------|-----------|----------|----------|----------|
| **Level 5** | 复杂文本，熟悉和不熟悉话题 | 完全理解复杂论证，评估观点，比较不同意见 | 在复杂文本中进行广泛推理，基于深层含义推断 | 从语境中推断复杂词汇含义，识别词汇的细微差别 |
| **Level 4** | 相当复杂的文本，熟悉话题 | 理解相当复杂的文本，追踪论证发展 | 在相当复杂的文本中进行推理 | 从语境中推断词汇含义 |
| **Level 3** | 简单文本，尤其是熟悉话题 | 理解简单文本，追踪论证的部分发展 | 在简单文本中进行基本推理 | 在简单语境中推断词汇含义 |
| **Level 2** | 简单文本的部分内容 | 如果话题和词汇熟悉，理解简单文本的部分 | 推理能力有限 | 推断能力有限 |
| **Level 1** | 简单文本的部分内容 | 如果话题和词汇熟悉，理解简单文本的部分 | 推理能力非常有限 | 在非常简单的语境中推断词汇 |

**维度2：写作能力 (Writing)**

| 等级 | 内容质量 | 语言范围 | 准确性 | 组织结构 |
|------|----------|----------|--------|----------|
| **Level 5** | 内容相关且广泛，吸引读者，有创意 | 使用广泛的句式结构，准确恰当 | 标点和语法准确，传达意义清晰 | 完全连贯，段落划分有效，句子间和段落间的衔接复杂且成熟 |
| **Level 4** | 内容相关，详细，吸引读者，有创意 | 使用一系列句式结构 | 标点符号和语法足够准确，错误不影响整体清晰度 | 大部分连贯，段落划分足够有效，大部分句子和段落之间衔接成功 |
| **Level 3** | 大部分内容相关，有些创意，熟悉情境下表现良好 | 使用一些更复杂的句式和常见词汇，准确使用 | 基本标点符号和一些基本语法结构准确 | 有些部分连贯且适合文体，有一些段落划分的证据，有些简单的句子间和段落间联系 |
| **Level 2** | 有些相关内容 | 使用简单句式结构和简单词汇，准确使用 | 基本语法，常见词汇适当拼写 | 当文体和文本类型简单熟悉时，结构可以辨认，有一些段落划分的证据 |
| **Level 1** | 一两个相关要点 | 使用几个简单可理解的句子 | 有一些简单词汇适当使用的例子 | 当文体和文本类型简单熟悉时，结构可以辨认 |

**维度3：听力理解 (Listening)**

| 等级 | 语速适应 | 理解范围 | 态度识别 | 信息提取 |
|------|----------|----------|----------|----------|
| **Level 5** | 接近自然语速 | 理解口语的字面和比喻意义 | 通过语调和重音评估观点，识别说话者的态度和意图 | 在熟悉口音和广泛情境中理解复杂信息 |
| **Level 4** | 中等语速 | 理解口语的字面和比喻意义 | 在相当复杂的文本中评估观点，通过明显的语调和重音特征识别说话者的态度和意图 | 在熟悉口音和大多数情境中理解相当复杂的文本 |
| **Level 3** | 中等语速 | 理解口语的字面意义 | 在简单文本中识别明确表达的观点和说话者的态度和意图 | 在熟悉口音和熟悉情境中理解简单口语 |
| **Level 2** | 中等语速 | 理解简单口语 | 识别和提取简单可预测的信息 | 在熟悉口音和熟悉情境的短文本中理解简单口语 |
| **Level 1** | 中等语速 | 在短文本中理解简单口语 | 识别和提取简单可预测的信息 | 在熟悉口音的熟悉主题上识别简单信息 |

**维度4：口语表达 (Speaking)**

| 等级 | 流利度 | 发音与语调 | 词汇使用 | 语言结构 | 互动能力 |
|------|--------|-----------|----------|----------|----------|
| **Level 5** | 流利表达广泛的观点，极少犹豫 | 适当投射声音，发音清晰，语调自然流畅，语调成熟 | 使用令人印象深刻的广泛词汇 | 使用多样且高度准确的语言模式，必要时有效改述 | 全面使用策略，熟练地发起和维持交流，有目的性地回应他人 |
| **Level 4** | 流利表达一系列观点，偶尔犹豫 | 令人满意地投射声音，发音清晰，流利自然 | 广泛使用词汇，大部分恰当 | 使用多样且大部分准确的语言模式，只有轻微失误，不妨碍交流 | 使用广泛的适当策略，发起和维持交流，适当回应他人 |
| **Level 3** | 流利地使用一系列简单常见表达 | 声音投射不佳可能造成困难，发音清晰，说话相当流利 | 使用多样且大部分恰当的词汇 | 使用一般准确的语言模式，偶尔有错误 | 使用简单策略，偶尔参与，被提示时回应 |
| **Level 2** | 可理解地表达简单观点 | 声音投射不佳，发音简单声音足够好，犹豫 | 从有限/熟悉范围中适当使用 | 使用适当的简单模式，偶尔有错误 | 主要通过程式化表达使用简单策略 |
| **Level 1** | 用简单可理解的句子表达观点 | 发音简单声音和常见单词足够准确，语调在最熟悉的单词和短语中适当 | 使用简单有限的词汇 | 使用简单模式，频繁错误 | 有限参与，需要提示才能回应 |

**官方数据来源**：
- 香港考评局《香港中学文凭考试英语科水平参照等级描述》
- HKEAA DSE English Language Subject Level Descriptors

### 3.8 写作练习模块（MVP低成本方案）

DSE英语写作分为 **Paper 2 Writing**，包含两部分：
- **Part A**：情景写作（必答，建议作答时间约40分钟，字数约200-300字）
  - 常见题型：书信、报告、便条、演讲辞、文章投稿、社论
- **Part B**：议题写作（三选一，建议作答时间约60分钟，字数约400-600字）
  - 常见题型：议论文、说明文、记叙文、散文

#### MVP阶段评分方案：范文对比 + 自评清单

**MVP阶段不使用AI评分**（成本控制），改用以下替代方案：

**业务流程**：

```
进入写作 → 选择考试部分（Part A/Part B）→ 选择题目（真题/模拟）
    ↓
    ┌──────────────────────────────────┐
    │  题目要求展示（情景/议题）       │
    │  字数要求：XX-XX                │
    │  建议时间：XX分钟               │
    │  题目原文：...                 │
    └──────────────────────────────────┘
    ↓
    用户输入作文 → 字数统计检查
    ↓
    提交后显示：
    ┌──────────────────────────────────┐
    │  📊 基础检查                     │
    │  字数：XXX / 要求：200-300      │
    │  用时：XX分钟                   │
    │  完成状态：✅ 已达标 / ❌ 未达标 │
    │                              │
    │  📝 自评清单（请对照检查）：     │
    │  ☐ 是否回答了题目所有要求？      │
    │  ☐ 文章结构是否清晰（开头/主体/结尾）？
    │  ☐ 是否使用了适当的连接词？      │
    │  ☐ 是否有语法或拼写错误？        │
    │  ☐ 词汇是否丰富多样？            │
    │                              │
    │  📖 参考范文（3篇不同等级）：    │
    │  [查看Level 3范文]              │
    │  [查看Level 4范文]              │
    │  [查看Level 5范文]              │
    │                              │
    │  💡 写作技巧提示：               │
    │  - 此类题目常用开头句式          │
    │  - 高分词汇推荐列表              │
    │  - 常见错误警示                  │
    └──────────────────────────────────┘
```

**范文库设计**：
- 每道题目提供3篇参考范文（Level 3/4/5）
- 范文包含批注：亮点词汇、句型结构、得分点说明
- 学生可以对比自己的作文与范文的差距

**自评清单（DSE官方标准）**：
| 维度 | 自评问题 | 权重 |
|------|---------|------|
| 内容 | 是否完全回应题目要求？观点是否清晰？ | 40% |
| 组织 | 段落划分是否合理？逻辑是否连贯？ | 30% |
| 语言 | 词汇是否准确丰富？语法错误是否很少？ | 30% |

**学生自评后**：
- 系统记录自评结果（不强制要求准确）
- 保存作文原文到个人历史
- 可多次修改并对比进步

#### MVP+1阶段：AI评分升级

**MVP+3将接入AI评分**（ArkCode API）：
- 自动从内容、组织、语言三维度评分
- 逐句语法检查和修正建议
- 词汇升级推荐
- 预计成本：¥500-1000/月（按量计费）

### 3.10 错题本模块

**业务流程**：

```
进入错题本
  ├─ 支持按题型/知识点筛选
  ├─ 显示今日需复习错题（根据艾宾浩斯曲线计算）
  └─ 开始复习
       ↓
       错题重现 → 再做一次
       ↓
       做对 → 进入下一阶段复习/从错题本移除
       做错 → 重置复习阶段，下次间隔缩短
```

**艾宾浩斯间隔重复算法**：

基于艾宾浩斯遗忘曲线，设置8个复习阶段：

| 阶段 | 间隔时间 | 记忆保持率目标 |
|------|---------|---------------|
| 第1次复习 | 5分钟后 | 短期记忆巩固 |
| 第2次复习 | 30分钟后 | 短期→中期过渡 |
| 第3次复习 | 12小时后 | 中期记忆建立 |
| 第4次复习 | 1天后 | 长期记忆开始 |
| 第5次复习 | 2天后 | 长期记忆巩固 |
| 第6次复习 | 4天后 | 深度记忆 |
| 第7次复习 | 7天后 | 永久记忆形成 |
| 第8次复习 | 15天后 | 完全掌握 |

**算法逻辑**：
```
错题录入：
1. 首次做错 → 加入错题本，设置阶段=1，下次复习时间=当前+5分钟

复习调度：
2. 每日凌晨计算今日需复习错题
   今日复习列表 = 所有 next_review ≤ 今天的错题

复习反馈：
3. 用户复习错题：
   IF 做对：
     stage = stage + 1
     IF stage > 8：从错题本移除（已掌握）
     ELSE：按阶段设置下次复习时间
   IF 做错：
     stage = max(stage - 2, 1)  【回退2个阶段，最少到1】
     设置下次复习时间为5分钟后（立即强化）

难度调整：
4. 连续2次在同一阶段做错 → 标记为"顽固错题"，增加提示和讲解
5. 同一知识点错题>3道 → 推送相关知识点练习
```

**勤奋学模式中的错题复习**：
- 每日固定5分钟错题复习时间
- 优先复习阶段1-3的近期错题（遗忘临界点）
- 复习完成自动进入当日学习统计

### 3.13 计分逻辑与自适应调整算法

#### 计分逻辑：边际递减得分制

**设计原则**：
- 每个模块有固定时间和默认题量
- 学生可在固定时间内完成更多题目
- 多做题且正确率达标可获额外加分，但边际效果递减
- 督促学生尽力多完成题目的同时，也要保证足够的思考时间和准确率

**基础得分计算**：
```
基础得分 = 基础分 × 正确率系数 × 难度系数

其中：
- 基础分 = 60分（完成默认题量即可获得）
- 正确率系数 = min(正确率 / 0.6, 1.0)  【正确率60%为基准线】
- 难度系数 = 当前等级对应系数（Level 1: 1.0, Level 5: 1.5）
```

**额外加分（边际递减）**：
```
额外加分 = 额外做题数 × 正确率奖励系数 × 边际递减系数

其中：
- 额外做题数 = 实际做题数 - 默认题量
- 正确率奖励系数：
  * 正确率 ≥ 80%：1.0（全额奖励）
  * 正确率 60-79%：0.7（部分奖励）
  * 正确率 < 60%：0（无奖励）
- 边际递减系数 = 1 / (1 + 0.1 × 额外做题数)
  【每多做1题，边际效益递减10%】

示例：
- 词汇模块：默认15题，学生完成25题（额外10题）
- 正确率85%（≥80%，系数1.0）
- 边际递减系数 = 1 / (1 + 0.1 × 10) = 0.5
- 额外加分 = 10 × 1.0 × 0.5 = 5分
- 模块总分 = 60 + 5 = 65分

再做多10题（共额外20题）：
- 边际递减系数 = 1 / (1 + 0.1 × 20) = 0.33
- 额外加分 = 20 × 1.0 × 0.33 = 6.6分（边际效益明显降低）
```

**模块满分**：100分
- 基础分：60分（完成默认题量+60%正确率）
- 额外分：最高40分（需做大量题目且保持高正确率）

**每日总分计算**：
```
每日总分 = Σ(各模块得分) / 模块数

轻松学（4个模块）：
总分 = (单词得分 + 词汇得分 + 语法律分 + 阅读得分) / 4

勤奋学（5个模块，含错题复习）：
总分 = (单词得分 + 词汇得分 + 语法律分 + 阅读得分 + 错题得分) / 5
```

**每日总分满分**：100分

#### 难度系数体系

**难度等级与系数对应表**：

| 能力等级 | 难度系数范围 | 说明 |
|---------|-------------|------|
| Pre Level | 0.8 - 1.0 | 小学阶段，预备级 |
| Level 1 | 1.0 - 1.9 | DSE Level 1，从1.0开始 |
| Level 2 | 2.0 - 2.9 | DSE Level 2 |
| Level 3 | 3.0 - 3.9 | DSE Level 3 |
| Level 4 | 4.0 - 4.9 | DSE Level 4 |
| Level 5 | 5.0 - 5.9 | DSE Level 5*，最高5.9 |

**难度提升机制**：
- 入学测试定级后，默认从该等级的最低值开始（如Level 1 → 1.0）
- 通过每日学习和良好表现，难度系数逐步提升（每次+0.05至+0.1）
- 达到当前等级上限（如1.9）后，自动晋升到下一等级
- 学生可随时重做入学测试重新定级

#### 自适应调整规则

**推送难度计算**：
```
题目难度范围 = [当前难度系数 - 0.3, 当前难度系数 + 0.2]

约束条件：
1. 不会推送远超出当前水平的题目（上限：当前难度 + 0.5）
2. 优先推送薄弱知识点的题目
3. 新知识点题目占20%，巩固题目占80%
```

**"远超出当前水平"判定**：
- 题目难度 > 用户当前难度系数 + 0.5 → 判定为"过难"
- 例如：用户当前2.3，不会推送难度 > 2.8 的题目
- 过难的题目会导致挫败感，影响学习动力

**每日调整规则**：
1. **正确率 < 60%**：降低难度系数（-0.1），增加基础题比例
2. **正确率 60-80%**：保持当前难度，微调知识点分布
3. **正确率 > 80% 且完成额外题目**：提升难度系数（+0.05至+0.1）
4. **错题知识点**：根据艾宾浩斯记忆曲线安排复习
5. **做题数量 < 默认题量**：提示"多做题可获得更高分数"
6. **连续3天正确率>85%**：建议重做入学测试，可能跳级

| 级别 | 词汇量 | 对应难度 | 对应年级 | 说明 |

**总词汇量**：约 7000词，完全覆盖DSE要求 + 小学预备阶段

- **P3（小三）**：开始预备级，逐步积累

### DSE考试结构与难度对应

基于DSE官方考试设计，我们的难度分级与DSE试卷结构对应：

**DSE英语科公开考试结构**：
- **Paper 1 Reading（阅读）**：20% - Part A + Part B1/B2
- **Paper 2 Writing（写作）**：25% - Part A + Part B1/B2
- **Paper 3 Listening（听力）**：30% - Part A + Part B1/B2
- **Paper 4 Speaking（口语）**：10% - Part A + Part B
- **SBA（校本评核）**：15%

**分级难度设计（B1 vs B2）**：

| 试卷部分 | B1难度 | B2难度 | 对应等级 |
|---------|--------|--------|----------|
| **Part A**（必答） | 基础-中等 | 基础-中等 | Level 1-3 |
| **Part B1**（选答） | 中等 | - | 最高Level 4 |
| **Part B2**（选答） | - | 难 | Level 3-5** |

**关键设计**：
- DSE采用**分级评卷**设计：选答B1的考生最高只能获得Level 4
- 选答B2的考生可以获得Level 1-5**的全部分数
- 这反映了DSE对高难度文本的理解和应用能力要求

**我们的题库设计对应**：
- **Part A题目**：对应Level 1-3（日常练习，轻松学模式）
- **Part B1题目**：对应Level 3-4（中等难度，勤奋学模式Part B）
- **Part B2题目**：对应Level 4-5（高难度，专项练习和挑战模式）

**历年合格分数线参考**（2012-2024数据）：

| 年份 | Level 5** | Level 5* | Level 5 | Level 4 | Level 3 | Level 2 |
|------|-----------|----------|---------|---------|---------|---------|
| 2024 | 84 | 78.7 | 65 | 53 | 35 | - |
| 2023 | 75 | 81 | 74.9 | 50 | 28 | - |
| 2022 | 63 | 68.4 | 57 | 47 | 26 | - |

**说明**：
- Level 5**（5星星）：前10%考生
- Level 5*（5星星）：前1-2%考生
- Level 3：合格线，大学入学基本要求
- Level 2：未达到合格标准

## 9. 技术架构

### 整体架构
- **前端**：React 18 + TypeScript + Tailwind CSS + Shadcn UI
  - 原因：组件丰富，开发快，样式美观，适合MVP
  - Tailwind Utility-first 快速开发

- **后端**：Node.js + Express / Next.js Full Stack
  - 原因：全JS栈，开发效率高，前后端可以一人维护
  - 选择Next.js = 前端+后端一体化，部署简单

- **数据库**：SQLite（MVP阶段）→ PostgreSQL（后续）
  - MVP：SQLite文件数据库，零成本，足够支撑初期用户
  - 后续：用户量增长后迁移到PostgreSQL

- **缓存**：Redis (可选，MVP可以不用，后期加)

- **AI能力（MVP阶段不使用，后续迭代接入）**：
  - ~~作文/口语内容评分：调用ArkCode / GPT-4o API~~ → **MVP+1/MVP+2阶段接入**
  - ~~ASR语音识别：OpenAI Whisper (本地部署或API)~~ → **MVP+2阶段接入**
  - **TTS语音合成**：Web Speech API（浏览器原生，零成本）✅ MVP使用
  - ~~发音评分：可以用Whisper+比对，或第三方API~~ → **MVP+2阶段接入**

**MVP阶段技术栈（低成本）**：
- 零成本AI：浏览器Web Speech API（TTS）
- 低成本后端：SQLite（后续迁移PostgreSQL）
- 无Redis：MVP阶段不需要缓存
- 自托管：单台腾讯云轻量服务器

**MVP+1/MVP+2阶段扩展**：
- AI评分：ArkCode API（作文）、Whisper+GPT-4o（口语）
- 数据库：迁移到PostgreSQL
- 缓存：Redis（PK对战、排行榜）

- **部署**：
  - 服务器：腾讯云Ubuntu，已经购买
  - Docker + Docker Compose 容器化部署
  - Nginx 反向代理 + SSL证书 (Let's Encrypt)
  - CI/CD：GitHub Actions → 代码push自动构建部署

|--------|------|------|
| 200 | 成功 | 请求成功处理 |
| 400 | 参数错误 | 缺少必填参数或格式错误 |
| 401 | 未授权 | Token无效或过期 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 资源不存在 | 请求的资源不存在 |
| 409 | 冲突 | 资源已存在（如重复注册）|
| 422 | 业务逻辑错误 | 业务规则校验失败 |
| 429 | 请求过于频繁 | 限流触发 |
| 500 | 服务器错误 | 服务器内部错误 |

**认证机制**：JWT Token
- 登录成功后返回 `accessToken` 和 `refreshToken`
- 请求头携带：`Authorization: Bearer <accessToken>`
- Token有效期：accessToken 2小时，refreshToken 7天

## 13. 编码规范

1. **TypeScript**：所有代码必须加类型，禁止 `any`
2. **命名**：驼峰命名，函数名动词开头，如 `getUser()`
3. **组件**：一个文件一个组件，组件名大驼峰
4. **API**：统一返回格式 `{code: number, data: T, message: string}`
5. **错误处理**：所有异步操作必须捕获错误
6. **Git**：每次提交一个清晰的功能，commit message 语义化

## 14. 部署方案

### MVP阶段部署（低成本方案）

#### 基础设施
- **服务器**：腾讯云轻量服务器（2核4G，约¥50-100/月）
- **域名**：注册域名，解析到服务器IP
- **SSL**：Let's Encrypt 免费证书
- **数据库**：SQLite本地文件（零成本，无需额外配置）
- **AI服务**：浏览器Web Speech API（零成本）

**月成本预估：¥50-100**

#### Docker Compose 配置（MVP简化版）
```yaml
version: '3.8'
services:
  app:
    build: .
    restart: always
    ports:
      - "3000:3000"
    environment:
      - JWT_SECRET=your-secret-key
      - NEXTAUTH_URL=https://your-domain.com
    volumes:
      - ./prisma/dev.db:/app/prisma/dev.db  # SQLite持久化
      - ./uploads:/app/uploads              # 文件上传目录
  nginx:
    image: nginx:alpine
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./certbot/conf:/etc/letsencrypt
```

**说明**：
- 无PostgreSQL：使用SQLite，单文件数据库
- 无Redis：MVP阶段不需要缓存
- 无AI服务：使用浏览器Web Speech API

## 16. 风险与注意事项

### MVP阶段风险

1. **版权**：MVP用公开真题验证，商业化后需要转自研AI出题
2. **无AI评分体验**：写作/口语使用自评清单，用户体验可能不如AI评分直观
   - **应对**：提供详细范文/示范录音对比，清晰的自评指引
3. **音频存储**：口语录音存储在服务器，需控制存储成本
   - **应对**：限制录音时长和保存数量，定期清理旧录音
4. **成本控制**：MVP阶段成本很低（¥50-100/月），主要是服务器费用
   - **优势**：零AI服务成本，SQLite零数据库成本

### 后续迭代风险

5. **AI评分准确性**：MVP+1/MVP+2接入AI后，需要持续校准与DSE评分标准对齐
6. **AI服务成本**：接入AI后成本增加，需监控API调用量
   - **应对**：设置每日/每月调用上限，超出后提示用户
