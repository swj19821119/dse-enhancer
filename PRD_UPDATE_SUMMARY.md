## 🆕 v2.5 更新（2026-03-25）

### 更新原因
成本控制：移除AI服务，改用低成本替代方案

### 1. MVP范围调整（移除AI服务）

**从MVP移除**：
- ~~AI作文评分（ArkCode API）~~ → MVP+3阶段（3-6个月后）接入
- ~~AI口语评分（Whisper + GPT-4o）~~ → MVP+3阶段（3-6个月后）接入

**AI评分升级时机**：
- 推迟到MVP+3（上线后3-6个月）
- 基于用户反馈和收入情况决定是否接入
- 如AI成本过高，保留范文对比/录音对比作为备选方案

**MVP阶段替代方案**：

#### 写作练习 - 范文对比 + 自评清单
- 每道题目提供3篇参考范文（Level 3/4/5）
- 结构化自评清单（内容/组织/语言三维度）
- 字数统计和基础检查
- 写作技巧和高分词汇提示
- 保存作文历史，对比进步

#### 口语练习 - 录音对比 + 自评清单
- 每道题目提供3个等级示范录音
- 本地录音保存和回放
- 结构化自评清单（发音/流利度/内容/互动）
- 口语技巧和高分表达提示
- 反复练习，对比进步

### 2. 技术架构调整

**MVP阶段技术栈（零AI成本）**：
- **数据库**：SQLite（零成本）→ 后续迁移PostgreSQL
- **语音**：Web Speech API（浏览器原生，零成本）
- **后端**：单台轻量服务器（¥50-100/月）
- **无Redis**：MVP阶段不需要

**成本对比**：

| 阶段 | 月成本 | 说明 |
|------|--------|------|
| **MVP** | ¥50-100 | SQLite + Web Speech API，无AI服务 |
| **MVP+1** | ¥100-200 | 题库扩容，阿里云TTS（听力）|
| **MVP+2** | ¥200-500 | 题库扩容至2000+题，PK对战功能 |
| **MVP+3** | ¥500-1500 | 接入AI评分（ArkCode+Whisper+GPT）+ PostgreSQL + Redis |

### 3. 部署方案简化

**MVP Docker Compose**：
- 仅包含app + nginx
- SQLite文件持久化
- 无PostgreSQL、无Redis、无AI服务

**后续扩展**：
- MVP+2阶段再引入完整架构

### 4. MVP题库建设策略（v2.5.1补充）

**目标**：满足3-5个试用用户2周（14天）学习用量

**题目数量建议**：

## v2.4 更新（2026-03-25）

### 更新原因
完善香港学制支持、访客模式细节、计分逻辑和API设计

### 1. 年级选择改为香港学制

**新设计**：
- 年级选项：P1-P6（小学）, F1-F6（中学）
- P1-P6对应Pre Level（0.5-1.0），为DSE打基础
- F1-F6对应DSE Level 1-5（1.0-5.9）

**入学测试按年级分配难度**：
- P1-P3: Pre Level (0.5-0.9)
- P4-P6: Pre Level (0.7-1.0)
- F1-F3: Level 1-2 (1.0-2.0)
- F4-F6: Level 1-5 (1.0-5.0)

### 2. 访客模式完善

**数据存储**：
- 入学测试答案和结果 → localStorage
- 单次学习数据 → 内存（页面关闭丢失）
- 访客与注册用户共用相同学习入口

**访客流程**：
1. 选择年级 → 入学测试 → 显示等级
2. 进入每日学习（与注册用户相同界面）
3. 首次学习结束 → 数据不保存 → 引导注册

### 3. 计分逻辑：边际递减得分制

**核心原则**：
- 固定时间 + 默认题量
- 多做题且正确率达标可获额外加分
- 边际效果递减（防止盲目刷题）

**计算公式**：
```
基础分 = 60分（完成默认题量+60%正确率）
额外分 = 额外做题数 × 正确率系数 × 边际递减系数
边际递减系数 = 1 / (1 + 0.1 × 额外做题数)
模块满分 = 100分
```

**难度系数体系**：
- Level 1: 1.0-1.9
- Level 5: 5.0-5.9
- 默认从当前等级最低值开始
- 可重做入学测试重新定级

### 4. 学习模式时间调整

| 模式 | 总时长 | 模块安排 |
|------|--------|----------|
| 轻松学 | 40分钟 | 单词10'+词汇10'+语法10'+阅读10' |
| 勤奋学 | 60分钟 | 轻松学40'+Part B 15'+错题复习5' |

### 5. 错题本：艾宾浩斯记忆法

**8阶段复习间隔**：
1. 5分钟后
2. 30分钟后
3. 12小时后
4. 1天后
5. 2天后
6. 4天后
7. 7天后
8. 15天后（掌握）

**做错处理**：
- 回退2个阶段（最少到1）
- 下次复习时间设为5分钟后

### 6. "远超出当前水平"判定

**判定标准**：
```
题目难度 > 用户当前难度系数 + 0.5 → 判定为"过难"
```

**示例**：
- 用户当前2.3，不会推送难度 > 2.8 的题目
- 防止挫败感，保护学习动力

### 7. 完整API设计

**统一响应格式**：
```json
{
  "code": 200,
  "data": {},
  "message": "success"
}
```

**包含内容**：
- 认证机制（JWT Token）
- 错误码定义（10种常见错误）
- 请求/响应参数示例
- 10+个API接口详细设计

### 8. 数据库表调整

**User表更新**：
- grade: VARCHAR(10) 支持P1-P6, F1-F6
- current_level/target_level: FLOAT (0.5-5.9)
- study_mode: ENUM (relaxed/diligent)

**Question表**：
- difficulty: FLOAT (0.5-5.9)

**错题本表**：
- 新增stage/next_review/is_mastered字段

**移除表**（MVP阶段）：
- ~~vocabulary_pk_games~~ (MVP+2)
- ~~audio_records~~ (MVP+2)

### 9. 术语统一

| 术语 | 说明 |
|------|------|

| 轻松学 | 40分钟基础学习模式 |

## v2.3 更新（2026-03-25）

### 更新原因
优化无注册体验流程和计分逻辑

### 1. 无注册体验流程调整

**原设计**：
- 访客只能做5题快测 + 每种题型体验3题
- 固定体验题组

**新设计**：
- 访客可完整进行入学自适应测试（15题）
- 测试完成后可进入每日学习（随机题目）
- 学习结果不保存，无法进行自适应学习
- 在无学习进度状态下，可进行每日学习

**注册引导话术**：「注册后可保存学习进度，AI会根据你的弱项智能推送练习，帮你更快提分！」

### 2. 计分逻辑调整：效率得分制

**原设计**：
- 固定题量，按得分计算
- 每日总分 = Σ(模块得分 × 难度权重)

**新设计**：
- 固定时间，不限题量
- 模块得分 = 正确率 × 做题数量 × 难度系数 × 时间效率系数
- 做题越多，正确率越高，得分越高

**激励逻辑**：
- 做题越多，得分上限越高
- 正确率越高，得分越高
- 鼓励在保证正确率的前提下多做题

### 3. 数据库表结构调整

**DailyStudyRecord表新增字段**：
- `total_questions` - 今日总做题数
- `total_correct` - 今日总答对数
- `total_accuracy` - 今日总正确率
- 各模块新增 `*_questions` 和 `*_correct` 字段

### 4. 每日总结报告格式调整

**原格式**：固定满分制（如 词汇：XX/25）

**新格式**：显示做题数量和正确率
```
词汇：XX题 ✓XX 正确率XX% 得分XX
```
