#### 任务1: 入学测试模块
**状态**: ⏳ 待开发  
**预计工时**: 2-3天  
**依赖**: 数据库表已创建

**需求**:
- 15题自适应测试（根据年级F1-F6选择难度）
- 实时难度调整（答对+0.1难度，答错-0.1难度）
- 计算最终等级（Level 1-5，含小数位如2.3）
- 显示能力报告（各维度得分）

**技术要点**:
```typescript
// 需要实现的核心算法
interface PlacementTest {
  currentDifficulty: number;  // 当前题目难度 1.0-5.0
  correctCount: number;
  wrongCount: number;
  questions: Question[];
}

// 自适应调整逻辑
function adjustDifficulty(isCorrect: boolean, current: number): number {
  const delta = isCorrect ? 0.1 : -0.1;
  return Math.max(1.0, Math.min(5.0, current + delta));
}
```

**数据库表**: `placement_sessions`, `placement_answers`

**验收标准**:
- [ ] 用户可以选择年级（F1-F6）
- [ ] 15题测试流程完整
- [ ] 难度根据答题正确率动态调整
- [ ] 最终计算等级（如Level 2.3）
- [ ] 生成能力报告页面

#### 任务4: 计分与自适应调整
**状态**: ⏳ 待开发  
**预计工时**: 2-3天  
**依赖**: 任务2完成

**需求**:
- 边际递减计分制（详见PRD 3.13节）
- 基础分60分 + 额外分（边际递减）
- 模块满分100分，每日总分100分
- 根据正确率调整明日难度

**计分公式**:
```
基础得分 = 60 × (正确率/60%) × 难度系数
额外加分 = 额外题数 × 正确率奖励 × 边际递减系数
边际递减系数 = 1 / (1 + 0.1 × 额外题数)
```

**自适应规则**:
- 正确率 < 60%：明日难度-0.1，增加基础题
- 正确率 60-80%：保持当前难度
- 正确率 > 80%：明日难度+0.05至+0.1

**验收标准**:
- [ ] 每模块实时计分正确
- [ ] 额外做题边际递减效果正确
- [ ] 每日生成难度调整报告
- [ ] 数据保存到数据库

## 💡 开发建议

1. **优先完成P0任务**：核心学习闭环（入学测试→每日学习→计分→错题复习）
2. **使用Shadcn UI组件**：保持界面一致性
3. **遵循TypeScript**：所有新代码必须有类型定义
4. **数据库变更**：使用Prisma Migrate，不要直接修改schema
5. **测试**：关键函数需要单元测试
6. **代码提交**：遵循已有提交信息格式
